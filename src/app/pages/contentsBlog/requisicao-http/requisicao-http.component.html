Requisição HTTP e fetch API

Uma requisição HTTP - Hypertext Transfer Protocol - é um pedido enviado por um cliente/usuario a um servidor web para solicitar algum recurso. Esses recursos podem incluir diferentes tipos de dados como documentos HTML, imagens, arquivos de estilo, scripts, etc. O protocolo HTTP é usado para comunicação entre clientes e servidores na World Wide Web.

Os métodos de uma requisição HTTP indica a ação que o cliente deseja realizar. Os mais comuns são:

  1. GET para obter dados do servidor;
  2. POST para enviar dados para o servidor (geralmente utilizados para enviar dados de formulários e cadastros);
  3. PUT para atualizar um recurso existente no servidor;
  4. DELETE para excluir algo no servidor.

o Método Fetch é uma API moderna para fazer requisições HTTP em JavaScript. Ele fornece uma maneira mais flexível e poderosa para trabalhar com solicitações e respostas HTTP. A Fetch API retorna Promises, que são objetos que representam o resultado eventual (ou falha) de uma operação assíncrona, possuindo 3 estados: pendente, resolvida (fulfilled) e rejeitada. Isso simplifica o tratamento de requisições assíncronas e torna o código mais legível (facil de entender e escrever). Isso é especialmente útil quando você precisa realizar várias operações em sequência.

A resposta (Response) retornada por fetch() contém várias informações sobre a requisição, incluindo o corpo da resposta, status, cabeçalhos, etc. O método json() da Response retorna uma Promise que resolve para o corpo da resposta parseado como JSON.

O fetch oferece melhor suporte para trabalhar com cabeçalhos (headers) de requisição e resposta. É possivel definir e manipular headers de forma mais eficiente. Também suporta facilmente diferentes métodos HTTP citados acima.
